# Источник данных рейтинга ПКО-300

## Что здесь лежит

Файлы вида `PKO-YYYY-MM.xlsx` — выгрузки из ФНС по реестру профессиональных коллекторских организаций (ПКО) с финансовой отчётностью за последние 5 лет.

**Это единственный источник истины для рейтинга.** TS-файлы данных (`design/src/app/data/ratingData.ts`, `financeDynamic.ts`, `companyDetails.ts`) генерируются из этого xlsx скриптом `scripts/update_data.py`.

## Откуда берётся файл

Выгрузка из ФНС России. Содержит:
- Юр. данные ПКО (ИНН, ОГРН, название, регион, адрес, гендиректор, дата основания)
- Финансовые показатели за 5 лет: выручка, прочие доходы, расходы, чистая прибыль
- Балансовые показатели: финансовые вложения, дебиторская задолженность, собственный/заёмный капитал
- Рассчитанные метрики: CAGR выручки, рост фин. активов, D/E коэффициент

## Как обновить данные

1. Положить новый xlsx в эту папку с именем `PKO-YYYY-MM.xlsx`.
2. Прогнать `python3 scripts/update_data.py --dry-run` — посмотреть, что изменится.
3. Если ОК → `python3 scripts/update_data.py` (без флага) — реальный прогон.
4. `cd design && npm run build` — проверить сборку.
5. Закоммитить изменённые `data/source/PKO-*.xlsx` и `design/src/app/data/*.ts` в одном коммите.

## Частота обновления

По мере появления свежих выгрузок ФНС (ориентировочно раз в квартал).

## Логика ранжирования

Компании сортируются **по полю «Выручка + прочие доходы 2025»** (по убыванию). Это даёт честную картину размера бизнеса каждой ПКО, включая нетипичные источники дохода (продажа портфелей долгов, инвестиционные операции).

Альтернативные ранжирования (по чистой прибыли, по выручке) не используются — они либо вводят в заблуждение (компании с разовыми бухгалтерскими событиями попадают в топ), либо игнорируют значительную часть деятельности (прочие доходы для ПКО — это нередко основной источник).

## Что сохраняется при обновлении

Скрипт `update_data.py` **сохраняет ручные правки** из текущих TS-файлов:
- Поле `napka` (НАПКА — членство в ассоциации)
- `capitalAttraction` (привлечение капитала: облигации, займы)
- `fundraising` и `bonds` (детали по привлечению средств)
- `website` (берётся из текущего ts; xlsx используется только если в ts пусто)

Финансовые показатели, ранг, fullName, address, director, regionDate — всегда перезаписываются из источника.

## Известные особенности источника

ФНС-выгрузка иногда оставляет пустыми поля у новых компаний:
- «Краткое название» — скрипт делает fallback на «Полное название» через `build_short_from_full()` (форматирует как `ОФ ПКО "БРЕНД"`).
- «Регион», «Стаж», «Дата основания» — остаются пустыми, нужны ручные правки.

**Если бренд в Полном названии — длинная фраза** (например, у `2635261351` бренд = «АГЕНТСТВО ПО ВОЗВРАТУ ЗАДОЛЖЕННОСТИ», но на их сайте они называют себя «АВЗ») — после прогона скрипта поправить `name` в `ratingData.ts` и `financeDynamic.ts` вручную, или заполнить «Краткое название» в xlsx перед следующим прогоном.
