## GIT WORKFLOW (ОБЯЗАТЕЛЬНО)

### Правило 1: Никогда не работай на ветке main
- Ветка `main` — только для стабильного кода
- Прямые коммиты в `main` **ЗАПРЕЩЕНЫ**
- Мердж в `main` — только через Pull Request после подтверждения пользователя

### Правило 2: Ветки для каждой задачи
- Перед каждой задачей создай новую ветку:
  ```
  git checkout main
  git pull origin main
  git checkout -b feat/название-задачи
  ```
- Формат имён веток:
  - `feat/` — новая функциональность (feat/add-founder-data)
  - `fix/` — исправление ошибок (fix/wrong-inn-mapping)
  - `data/` — обновление данных (data/update-bonds-2024)
  - `refactor/` — рефакторинг (refactor/restructure-capital-sheet)

### Правило 3: Атомарные коммиты
- Закончил кусок работы → сразу коммить
- Один коммит = одно логическое изменение
- НЕ копить изменения, НЕ делать гигантские коммиты
- Формат коммит-сообщений:
  ```
  feat: добавлены данные учредителей для 102 компаний
  fix: исправлен тип привлечения для ВПК-Капитал
  data: обновлена структура капитала — лист D/E ratio
  ```

### Правило 4: Push + Pull Request
- После завершения задачи:
  ```
  git push -u origin feat/название-задачи
  gh pr create --title "..." --body "..."
  ```
- В PR описании указать: что сделано, какие файлы затронуты, что проверить
- **НЕ мерджить PR** без подтверждения пользователя

### Правило 5: Безопасность истории
- Если что-то сломалось → `git revert`, НИКОГДА не `git reset --hard`
- **ЗАПРЕЩЕНО:** `git push --force`, `git reset --hard`, удаление веток с данными
- История коммитов — священна, не переписывать

### Правило 6: Синхронизация перед работой
- Перед каждой новой задачей:
  ```
  git checkout main
  git pull origin main
  ```
- Только потом создавать новую ветку

## ПРАВИЛА РАБОТЫ С ДАННЫМИ

### Верификация (из глобальных правил)
- **Каждый факт должен быть подтверждён** источником
- **НЕ выдавать предположения за факты**
- **Тестировщик проверяет** до отправки результата
- Если факт не подтверждён → честно пометить «не подтверждено»

### Источники данных
- **ФССП** — реестр ПКО (547 компаний)
- **rusprofile.ru** — учредители, ЕГРЮЛ
- **MOEX** — облигации, ISIN, купоны
- **Сайты компаний** — страницы для инвесторов
- **НЕ использовать** непроверенные данные из поисковой выдачи

### Изменения в Excel файлах
- ВСЕГДА делать коммит ПЕРЕД внесением изменений (backup)
- ВСЕГДА делать коммит ПОСЛЕ внесения изменений
- В коммит-сообщении указывать: какой лист, какие столбцы, сколько строк изменено

## ЗАПРЕЩЕНО В ЭТОМ ПРОЕКТЕ

- Создавать файлы с суффиксами `_fixed`, `_final`, `_v2`
- Коммитить в `main` напрямую
- Мерджить PR без подтверждения
- Использовать `git push --force` или `git reset --hard`
- Добавлять непроверенные данные в основную БД
- Удалять историю git
