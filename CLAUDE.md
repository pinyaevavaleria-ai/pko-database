# PKO Database — Правила работы

## Git Workflow (ОБЯЗАТЕЛЬНО)

### Ветвление
- **НИКОГДА не работай напрямую на ветке `main`**
- Перед каждой задачей создавай новую ветку: `feat/название-задачи`
- Примеры: `feat/check-founders`, `feat/add-bonds-data`, `fix/wrong-inn`

### Коммиты
- Коммить атомарно — закончил кусок работы, сразу коммить
- Сообщения коммитов на русском или английском, по смыслу изменений
- Не коммить файлы с секретами (.env, токены, пароли)

### Pull Requests
- После завершения задачи — пушь ветку и создавай Pull Request
- Мерджи PR в main **ТОЛЬКО после подтверждения пользователя**
- Никогда не мерджи самостоятельно

### Откаты
- Если что-то сломалось — используй `git revert`
- **НИКОГДА не удаляй историю** (no force push, no reset --hard на main)

### Начало новой задачи
- Всегда переключайся на main: `git checkout main`
- Делай `git pull` перед созданием новой ветки
- Создавай ветку: `git checkout -b feat/название-задачи`

## Протокол качества данных

### Принцип: ТОЛЬКО подтверждённые факты
- Если данные не подтверждены на 100% — помечай "не подтверждено"
- Источник каждого факта должен быть указан
- Тестировщик проверяет данные ДО отправки пользователю

### Источники данных
- **ФССП** — реестр ПКО (547 компаний)
- **rusprofile.ru** — учредители, ЕГРЮЛ
- **MOEX** — облигации, купоны
- **Сайты компаний** — страницы для инвесторов
- **СПАРК / Контур.Фокус** — финансовая отчётность (нет доступа)

### АБСОЛЮТНЫЙ ЗАПРЕТ НА ЛОЖНУЮ ИНФОРМАЦИЮ
- **НИКОГДА не говори "готово", "работает", "можно использовать" — если не проверил лично**
- **НИКОГДА не давай ссылки/инструкции — если не проверил что они реально работают**
- **НИКОГДА не утверждай что интеграция работает — если не сделал тестовый запрос**
- Выдавать предположения за факты — ЗАПРЕЩЕНО
- Додумывать связи между компаниями — ЗАПРЕЩЕНО
- Писать "материнская компания финансирует" без доказательств — ЗАПРЕЩЕНО
- Использовать данные без указания источника — ЗАПРЕЩЕНО

### Правило верификации (ПЕРЕД каждым ответом)
1. Я утверждаю что-то? → Я это ПРОВЕРИЛ или ПРЕДПОЛАГАЮ?
2. Если ПРЕДПОЛАГАЮ → честно написать "не проверено, предположение"
3. Даю ссылку? → Открыть и убедиться что работает
4. Говорю "совместимо/работает"? → Проверить ВСЕ условия (доступ, формат, права)
5. Если не могу проверить → сказать прямо: "не могу проверить, нужно протестировать"
